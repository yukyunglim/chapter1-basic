0. 이클립스 -> Chap09 -> constructor 에 만들어져 있음

1. 생성자(constructure)
    1) 기본 문법
        <class_name>({argument_list}) { {statements} }      argument_list와 statements 는 있을수도 있고 없을 수도 있음
    
    2) 객체를 생성할 때 new 키워드와 함께 사용
    3) 생성자는 일반 함수처럼 기능을 호출하는 것이 아니고 
       객체를 생성하기 위해 new와 함께 호출됨.
    
    4) 생성자는 객체를 생성될 때 변수나 상수를 초기화 하거나 다른 초기화 기능을 수행하는 메서드를 호출함.
        => 즉, 멤버 변수를 초기화하는 역할(메서드와는 역할이 다름)!!!!!

    5) 생성자는 반환 값이 없고, 클래스 이름과 동일하게 정의(알파벳 대소문자도 구분!)
    6) 대부분의 생성자는 외부에서 접근 가능하지만, 필요에 의해 private으로 선언되는 경우도 있음.
    7) 생성자는 복수 개 있을 수 있음

2. 기본 생성자(default constructure)
    1) 클래스에는 반드시 적어도 하나 이상의 생성자가 존재
    2) 클래스에 생성자를 구현하지 않아도 new 키워드와 함께 생성자를 호출할 수 있음.
    3) 클래스에 생성자가 하나도 없는 경우 컴파일러가 생성자 코드를 넣어줌.(기본 생성자)
        - 예) public Student() {}
    4) 매개변수가 없음, 구현부가 없음

3. 생성자 오버로딩(Overloading)
    1) 여러가지 생성자를 정의
    2) 클래스에 생성자를 따로 구현하면 기본 생성자(default constructure)는 제공되지 않음 -> 적어도 하나 이상의 생성자가 존재하므로
    3) 생성자를 호출하는 코드에서 여러 생성자 중 필요에 따라 호출해서 사용할 수 있음
